# Report Design Follow-up Sprint

## Purpose

Improve the paid PRO and ELITE report experience after the payment-flow launch.
This is a report-quality and presentation sprint. It must not change entitlement,
checkout, payment verification, or the Resume Engine's payment gate.

## Current Defects To Address

- PRO reports are too text-heavy and do not yet match the quality of the marketing experience.
- The Executive Summary contains a grammar defect in some generated reports.
- Several sections repeat the same finding, especially missing skills-section feedback.
- Keyword Insights are not distinct enough from the main improvement recommendations.
- The Priority Roadmap does not yet read like a practical, sequenced roadmap.
- Generic examples can imply metrics or achievements that are not supported by the source resume.
- Plan naming is not fully consistent across report, pricing, and product surfaces.

## Content Safety Standard

1. Every recommendation must be grounded in evidence present in the uploaded resume.
2. Any example that goes beyond source evidence must be explicitly labelled **Illustrative example**.
3. Never present invented achievements, employers, tools, scope, or metrics as candidate facts.
4. Reduce duplicated advice across the full report by approximately 35–45%.
5. Keep the voice direct, encouraging, and recruiter-oriented; avoid repetitive “AI-generated” framing.

## Proposed Paid Report Structure

1. Resume snapshot
2. ATS structure map
3. Recruiter 20-second scan
4. Keyword coverage
5. Evidence gaps
6. Priority matrix
7. Source-grounded before/after examples
8. Action plan

The PDF should adopt the same visual language as ResuNexx: editorial hierarchy,
restrained black/lime/orange accents, legible spacing, and clear decision points.

## Acceptance Criteria

- Executive Summary passes grammar review.
- Duplicate issue topics are deduplicated across sections.
- Keyword Insights have a distinct purpose and evidence model.
- Priority Roadmap specifies sequencing, effort, and expected communication benefit.
- All examples are source-grounded or labelled illustrative.
- PRO and ELITE reports visibly reflect their plan-specific entitlements.
- Plan names are standardized throughout the product.
- PDF output is visually consistent with the current ResuNexx design system.

## Validation

- Run the permanent benchmark dataset against the revised writer.
- Compare output length, duplicate-topic rate, grounding rate, and plan differentiation.
- Manually review representative graduate, technical, business, and career-change samples.

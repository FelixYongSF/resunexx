# Recruiter Report Writer 2.0

## Purpose

Report Writer 2.0 turns Resume Engine evidence into a premium written consultation.
It is not a separate analyzer and does not change payment, storage, or customer flow.

The analysis engine determines what is present. The report writer determines how to
explain the evidence, prioritize decisions, and coach the user toward a truthful edit.

## Product Principles

1. Start with credible value already present.
2. Describe the likely recruiter interpretation, not an abstract writing rule.
3. Separate evidence, interpretation, action, and expected communication benefit.
4. Give fewer recommendations and make each one executable.
5. Preserve facts. Never invent metrics, tools, employers, scope, or outcomes.
6. Use calm candor. Do not create urgency through fear or shame.
7. Keep the candidate in control of every suggested rewrite.
8. Explain uncertainty instead of presenting AI judgment as fact.

## Recommendation Standard

Every important recommendation should communicate:

- What was detected?
- Why does it matter?
- How may a recruiter or ATS interpret it?
- What should the user edit next?
- What communication benefit may follow?

These ideas should read as natural coaching, not as a repetitive five-label template.

## Premium Structure

- Recruiter First Impression: visible direction, strongest proof, and main uncertainty.
- Would a Recruiter Keep Reading: direct judgment, evidence, and the condition that
  would improve the judgment.
- Positive Standouts: resume-specific proof only.
- Hesitation Signals: decision-relevant uncertainty, not a repeated issue list.
- Five Most Important Changes: High, High, Medium, Medium, and Low impact.
- Section Feedback: what works and one focused next edit.
- Rewrite Examples: truthful, denser, and ready to adapt.
- Final Action Plan: ordered work that fits one focused editing session.
- Closing Note: credible confidence grounded in the candidate's existing foundation.

## Voice

- professional
- warm
- concise
- direct
- specific
- non-judgmental
- confidence-building

Avoid slogans, filler, robotic transitions, invented probabilities, and guarantees.

## Research Principles

The system draws on product principles demonstrated by current writing and career tools:

- audience and intent should shape suggestions
- explanations help users make informed edits
- reader reaction is more useful than grammar-only correction
- paid value comes from contextual, prioritized guidance
- AI output should remain reviewable and editable
- confidence and uncertainty should be visible

These principles are implemented independently. Competitor wording is not copied.

## Quality Measurement

Automated benchmark checks:

- actionability
- issue-topic duplication
- score calibration
- strong/medium/weak separation
- readiness distribution
- execution reliability

Human review checks:

- evidence grounding
- personalization
- recruiter realism
- emotional tone
- rewrite truthfulness
- perceived consultation value

## Future Direction

Report Writer should eventually receive a target job description and distinguish:

- baseline resume quality
- job-specific match
- evidence gaps
- communication gaps

These must remain separate scores. A resume cannot have a credible job-match score when
no target role or job description has been supplied.
